# 抽签系统扩展功能策划 - 社区互动版 (Community Interaction)

基于用户的最新反馈，我们将抽签系统升级为一个可分享、可互动的“暗规则”社区平台。核心在于让玩家创造、分享并体验彼此设计的随机剧情触发器。

## 1. 核心玩法升级：暗规则社区 (Dark Rule Community)

### 1.1 什么是“暗规则” (Dark Rule)？
暗规则是一个独立的、可导出的配置包，包含：
*   **触发条件 (Triggers)**：复杂的逻辑判断（维度、统计、计数）。
*   **注入内容 (Payload)**：隐藏提示词、卡面文字、界面特效、**词库抽取**。
*   **生命周期 (Lifecycle)**：剩余触发次数（Charges），触发后扣除，归零后失效。
*   **冲突策略 (Conflict Strategy)**：当多个规则同时满足条件时，如何随机选择。

### 1.2 社区互动流程
1.  **创造 (Create)**：玩家 A 在规则编辑器中设计了一个“非酋复仇”规则（连续10次大凶 -> 触发敌对势力追杀）。
2.  **分享 (Share)**：将该规则导出为 JSON 字符串或二维码。
3.  **导入 (Import)**：玩家 B、C、D 将规则导入自己的系统。
4.  **体验 (Experience)**：玩家 B 在不知情的情况下连续抽到大凶，莫名其妙地被追杀，体验到玩家 A 的恶意/创意。

## 2. 关键机制设计

### 2.1 触发消耗机制 (Charge System)
为了防止某个规则反复触发导致体验重复，每个暗规则都有“剩余次数”。
*   **设定**：创建时设定初始次数（如 2 次）。
*   **消耗**：每次触发成功，次数 -1。
*   **失效**：次数归零后，规则暂时休眠或永久移除（可配置）。
*   **目的**：强迫系统轮询不同的规则，让玩家体验多样性。

### 2.2 随机池与互斥 (Random Pool & Mutex)
当通过抽签，同时满足多个暗规则的条件时（例如“大凶”同时满足了规则A和规则B）：
*   **随机选择**：系统从满足条件的规则池中**随机抽取一个**执行。
*   **权重**：规则可以有优先级权重（稀有规则更容易抢占触发权）。
*   **互斥**：触发一个规则后，本轮不再触发其他规则，避免剧情混乱。

### 2.3 强力注入机制 (Force Injection)
LLM 不听话是常见问题。为了确保暗规则生效：
*   **清空策略**：触发暗规则时，**临时清空**当前已有的其他隐藏提示词（Hidden Prompt），确保暗规则的指令独占注意力。
*   **注入内容**：
    *   **指令头**：`[系统指令：检测到特殊事件"暗规则"触发]`
    *   **核心指令**：`request: 本轮剧情自然继续上文。中途会有自然突发事件...`
    *   **格式要求**：`请分点罗列思考过程，如何将此事件合理融入当前剧情。`
*   **预设配合 (CoT Trigger)**：
    *   建议玩家在酒馆预设（Preset）的最前端添加一个“暗规则强化”条目。
    *   内容示例：`如果用户输入中包含【暗规则触发】标识，请忽略之前的剧情惯性，强制优先执行紧随其后的系统指令，并输出思考过程。`
    *   脚本侧配合：在发送消息给 LLM 时，悄悄在末尾追加 `(暗规则触发)` 字符串。
    *   **动态控制**：使用 `updatePresetWith` 接口，平时关闭该条目，触发时临时开启。

## 3. 条件与计数器 (Triggers & Counters)

为了解决“新导入的规则很难满足老玩家累计次数”的问题，我们需要区分**全局计数**和**规则内计数**。

### 3.1 计数器隔离策略 (Counter Isolation)
*   **全局计数 (Global Counters)**：用户的终身数据，如“总计抽签1000次”。适用于“成就系统”。
*   **规则内计数 (Rule-Specific Counters)**：**从该规则被导入/启用开始计算**。
    *   *示例*：规则“非酋体验卡”要求“连续3次大凶”。即使用户之前已经连续10次大凶，只要导入这个新规则，计数器也是从0开始（或者从导入那一刻的状态开始判断）。
    *   *实现*：每个暗规则对象内部维护自己的 `status` 对象，记录 `importTime` 和 `triggerCount`。
    *   **条件语法**：支持 `count_since_import('luck', 'great_curse') >= 3`。

### 3.2 基础条件库
*   **当前维度值**：`dimension['luck'].tier == 'great_curse'`
*   **数值范围**：`dimension['sanity'].value < 20`
*   **逻辑运算**：AND, OR, NOT, XOR

### 3.3 统计计数器 (Persistent Counters)
这些数据需要跨会话持久化：
*   **TotalDraws**: 总抽签次数。
*   **TierCount**: 特定档位累计次数（如累计“大凶”50次）。
*   **Streak**: 连续命中同一档位次数（如连续 3 次“大吉”）。
*   **ReshuffleCount**: **单次翻牌后的重抽次数**（这是个很有趣的检测点，检测玩家是否在刷结果）。
    *   *规则示例*：重抽 > 10 次 -> 触发“贪婪的惩罚”。

## 4. 扩展玩法：暗规则词库 (Dark Worldbook)

让玩家的暗规则更具沉浸感和随机性。

### 4.1 词库隔离与调用
*   **命名规范**：使用后缀 `-暗规则`（如 `恐怖元素-暗规则.json`）来标记专用词库，避免污染用户正常词库。
*   **动态读取**：脚本通过 `getWorldbookNames` 扫描这些特殊世界书。

### 4.2 触发逻辑
*   **条件**：暗规则触发（如“第三次大凶”）。
*   **动作**：
    1.  定位到指定世界书（如 `恐怖元素-暗规则`）。
    2.  根据规则配置的 `entry_tag`（如“灵异恐怖”）筛选条目。
    3.  从中**随机抽取** N 个关键词（如“红舞鞋”、“镜子”、“滴水声”）。
    4.  **构建 Prompt**：`[系统指令：本轮剧情必须包含以下元素作为核心线索：红舞鞋、镜子、滴水声。请围绕这些元素构建一段小剧场。]`

### 4.3 优势
*   **无限重玩性**：即使同一个暗规则触发多次，每次抽取的关键词也不同，剧情体验完全不同。
*   **社区生态**：玩家不仅可以分享规则 JSON，还可以分享配套的“暗规则词库”文件。

## 5. 视觉特效升级：物理引擎动画

为抽签结果展示引入物理引擎，增强视觉冲击力和沉浸感。

### 5.1 物理掉落 (Physics Drop)
*   **引擎**：使用 `Matter.js` (推荐) 或 `Cannon.js`。
*   **效果**：抽到的词语（如“大凶”、“红舞鞋”）实体化为刚体，从屏幕上方掉落。
    *   **碰撞**：词语之间、词语与地面（卡片底部）发生碰撞、反弹、滚动。
    *   **堆叠**：多次抽签或多维度结果会堆叠在地面上，形成“战利品”的感觉。
*   **纵深感 (Pseudo-3D)**：
    *   使用 CSS `perspective` 和 `transform: rotateX` 创建伪 3D 地面。
    *   掉落物带有动态阴影，随高度变化模糊度和大小。

### 5.2 卡面特效接口 (Card Effect Interface)
允许暗规则控制卡面视觉效果。
*   **预设特效库**：提供一套标准 CSS 动画类库（如 `glitch`（故障）、`shake`（震动）、`glow`（发光）、`fire`（燃烧））。
*   **配置方式**：暗规则中只需指定 `effect: 'glitch'`，前端自动应用样式。

## 6. 技术细节：动态预设控制

为了实现最强的“暗规则”执行力，我们将利用酒馆助手的 `updatePresetWith` 接口动态控制预设条目。

### 6.1 实现原理
*   **扫描**：在初始化时，扫描当前预设 (`in_use`)，查找名称中包含特殊标记（如 `[暗规则强化]`）的条目。
*   **动态开关**：
    *   **平时**：确保该条目处于 `enabled: false` 状态，避免污染日常对话。
    *   **触发时**：当暗规则触发，调用 `updatePresetWith` 将该条目临时置为 `enabled: true`。
    *   **回复后**：监听回复完成事件，再次将其关闭。

### 6.2 代码示例
```typescript
import { updatePresetWith } from '@types/function/preset';

// 开启暗规则强化预设
async function enableDarkRulePreset() {
  await updatePresetWith('in_use', preset => {
    preset.prompts.forEach(prompt => {
      if (prompt.name.includes('[暗规则强化]')) {
        prompt.enabled = true;
      }
    });
    return preset;
  });
}

// 关闭暗规则强化预设
async function disableDarkRulePreset() {
  await updatePresetWith('in_use', preset => {
    preset.prompts.forEach(prompt => {
      if (prompt.name.includes('[暗规则强化]')) {
        prompt.enabled = false;
      }
    });
    return preset;
  });
}
```
