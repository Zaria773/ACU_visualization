# 抽签系统合规化重构计划

## 1. UI 合规度审查报告

基于用户反馈和代码审查，发现以下严重违规项：

| 违规类别 | 违规点 | 现状 (Current) | 规范要求 (Expected) | 修正方案 |
| :--- | :--- | :--- | :--- | :--- |
| **架构模式** | **弹窗层级** | 使用独立的 `DivinationSettingsDialog` 弹窗组件。 | 设置类功能应作为 `SettingsDialog` 的**子面板 (Sub-panel)**，通过 `currentPanel` 切换，保持单一弹窗上下文。 | 废弃独立弹窗，新建 `DivinationPanel.vue`，集成入 `SettingsDialog.vue`。 |
| **样式作用域** | **Scoped Style** | 使用了 `<style scoped>`。 | **禁止使用 scoped**。由于跨 iframe 架构，scoped 样式无法注入父窗口。 | 移除所有 scoped 样式。通用样式直接使用全局类，特有样式写入 `styles/components/` 下的 SCSS 文件。 |
| **CSS 类名** | **重复造轮子** | 定义了 `.acu-settings-input`, `.acu-edit-textarea`, `.acu-range-input` 等样式。 | 必须复用 `STYLE_INDEX.md` 中定义的 `.acu-settings-input`, `.acu-edit-textarea` 等通用类。 | 删除组件内的 CSS 定义，直接在 template 中使用标准类名。 |
| **交互逻辑** | **多层弹窗** | “运势配置”、“维度管理”等使用 `openLuckTierEditor` 打开新弹窗。 | 应优先使用**面板导航 (Breadcrumb Navigation)**，在同一弹窗内切换视图（如 `SettingsDialog` 的 `goToPanel` 逻辑）。 | 将子功能模块化为 Panel 组件，通过 `SettingsDialog` 的路由逻辑进行跳转。 |
| **UI 规范** | **输入控件** | 使用了文本输入框 (`input[type="number"]`)。 | **禁止出现输入框**，全部使用拖动条 (`input[type="range"]`)。 | 将所有数值输入改为滑块。 |
| **UI 规范** | **布局对齐** | 控件位置不统一，有的在左有的在右。 | **所有控件必须右对齐**。 | 使用 Flex 布局确保 `.acu-settings-control` 靠右。 |
| **UI 规范** | **行标题** | 有的行有提示语，有的没有。 | **所有行标题下必须有提示语 (Hint)**。 | 补全所有设置项的说明文案。 |
| **维度系统** | **结构混乱** | 运势和维度被拆分为两个独立系统。 | **运势也是一种维度**。维度应统一管理，支持折叠/展开，支持增删改档位、编辑提示词和调档位权重。 | 重构维度管理面板，统一运势和普通维度的交互逻辑。 |

## 2. 业务逻辑审查

| 检查项 | 现状 | 计划要求 | 结论 |
| :--- | :--- | :--- | :--- |
| **默认维度** | 默认为“时间”（过去/现在/未来）。 | 应为**“扭曲度”**（正常逻辑/天然合理）。 | **不合规**，需修改 `DEFAULT_DIMENSIONS`。 |
| **维度开关** | 仅有数据结构，未见 UI 控制。 | 需要支持 `enabled` 开关。 | **缺失**，需在 UI 中添加开关控件。 |
| **存储位置** | 使用 `getVariables({ type: 'script' })`。 | 必须使用 **`getVariables({ type: 'global' })`**。 | **不合规**，需修改 Store 中的存储逻辑。 |
| **图片上传** | 仅支持 URL 输入。 | 需要支持**图片上传**（IndexedDB 存储）和 URL 输入。 | **缺失**，需添加图片上传控件。 |

## 3. 全局重构计划与技术路线

### 阶段一：UI 架构重构 (Panelization)

**目标**：将抽签设置完全融入 ACU 标准设置体系，消除独立弹窗。

#### 3.1 标准子面板结构 (Standard Sub-panel)

合规的子面板（如 `DivinationPanel.vue`）应遵循以下结构：
*   **无弹窗外壳**：不包含 `.acu-modal`、`.acu-modal-header` 等元素，这些由父组件 `SettingsDialog` 统一渲染。
*   **无 Scoped 样式**：完全依赖全局样式。
*   **纯内容组件**：只包含设置项的具体内容。
*   **即时保存 (Auto-save)**：使用 `watch` 监听本地配置变化，并立即调用 Store 的更新方法。
*   **全滑块交互**：数值调整全部使用 `input[type="range"]`。

**代码模板示例**：
```vue
<template>
  <!-- 根元素无特殊类名，直接包裹内容 -->
  <div class="acu-divination-panel">
    <!-- 设置分区 -->
    <div class="acu-settings-section">
      <div class="acu-settings-title">
        <i class="fas fa-paint-brush"></i>
        外观设置
      </div>
      <div class="acu-settings-group">
        <!-- 设置行：直接使用通用类 -->
        <div class="acu-settings-row">
          <div class="acu-settings-label">
            抽词数量
            <span class="hint">每次抽签生成的词条数: {{ localConfig.drawCount }}</span>
          </div>
          <div class="acu-settings-control">
            <!-- 全滑块交互 -->
            <input
              v-model.number="localConfig.drawCount"
              type="range"
              min="1"
              max="10"
              step="1"
              class="acu-range-input"
            />
          </div>
        </div>
      </div>
    </div>

    <!-- 导航入口：跳转到三级页面 -->
    <div class="acu-settings-section">
      <div class="acu-settings-group">
        <div class="acu-nav-row" @click="emit('navigate', 'divination_dimension')">
          <div class="acu-settings-label">
            维度管理
            <span class="hint">管理运势和其他维度的档位、权重及提示词</span>
          </div>
          <div class="acu-settings-control">
            <i class="fas fa-chevron-right"></i>
          </div>
        </div>
      </div>
    </div>
  </div>
</template>

<script setup lang="ts">
import { ref, watch, reactive } from 'vue';
import { useDivinationStore } from '../../../stores/useDivinationStore';
import { klona } from 'klona';

const emit = defineEmits(['navigate']);
const store = useDivinationStore();

// 初始化本地配置副本
const localConfig = reactive(klona(store.config));

// 即时保存逻辑
watch(localConfig, (newVal) => {
  store.updateConfig(newVal);
}, { deep: true });
</script>

<!-- 禁止使用 <style scoped> -->
```

#### 3.2 增强 SettingsDialog 路由能力

为了支持“主设置 -> 抽签设置 -> 维度管理”的三级导航，需要增强 `SettingsDialog.vue` 的路由逻辑：

1.  **引入历史栈**：将 `currentPanel` 改为 `panelStack` 数组，或者保留 `currentPanel` 但增加 `historyStack`。
2.  **修改 `goToPanel`**：
    ```typescript
    const panelHistory = ref<PanelType[]>([]);

    const goToPanel = (panel: PanelType) => {
      panelHistory.value.push(currentPanel.value); // 保存当前面板到历史
      currentPanel.value = panel;
    };
    ```
3.  **修改 `goBack`**：
    ```typescript
    const goBack = () => {
      const prev = panelHistory.value.pop();
      if (prev) {
        currentPanel.value = prev;
      } else {
        currentPanel.value = 'main'; // 兜底
      }
    };
    ```
4.  **顶栏逻辑**：
    *   `SettingsDialog` 的顶栏（Header）是公共的。
    *   当 `panelHistory.length > 0` 时，显示“返回”按钮。
    *   点击“返回”按钮触发 `goBack`。

#### 3.3 组件拆分规划

*   `src/可视化表格/components/settings/divination/DivinationPanel.vue` (二级面板：抽签总览，含图片上传)
*   `src/可视化表格/components/settings/divination/DimensionManagerPanel.vue` (三级面板：统一管理运势和维度，支持折叠/展开，全滑块)
*   `src/可视化表格/components/settings/divination/WordPoolPanel.vue` (三级面板：词库管理)

### 阶段二：样式标准化 (Standardization)

**目标**：移除所有非标准样式，确保 100% 复用 Design Tokens。

1.  **清理代码**：
    *   在 `DivinationPanel.vue` 中，**不写任何 `<style>` 标签**。
    *   输入框使用 `<input class="acu-settings-input">`。
    *   文本域使用 `<textarea class="acu-edit-textarea">`。
    *   颜色选择器使用 `<input type="color" class="acu-color-input">` (需确认 `inputs.scss` 中是否有此定义，若无则添加通用类)。
2.  **补充通用样式**：
    *   如果发现现有通用类无法满足（例如垂直排列的设置行），在 `src/可视化表格/styles/layouts/settings.scss` 中添加 `.acu-settings-row.is-vertical`，而不是在组件里写。

### 阶段三：业务逻辑修正 (Correction)

**目标**：对齐策划文档。

1.  **Store 修正 (`useDivinationStore.ts`)**：
    *   修改 `DEFAULT_DIMENSIONS` 常量。
    *   修改 `loadFromScriptVariables` -> `loadFromGlobalVariables`。
    *   修改 `saveToScriptVariables` -> `saveToGlobalVariables`。
    *   确保 `SCRIPT_ID` 不再被用于配置存储。
2.  **功能补全**：
    *   在维度管理面板中，添加“启用/禁用”维度的开关 UI。
    *   添加图片上传逻辑（使用 `useBackgroundStorage` 或类似的 IndexedDB 机制）。
